Data usage disclaimer

What data does this project collect?

This project collects the following data from all participants:
- Your username
- Your password (encrypted using a hash function in storage)
- Your course codes and times, used for matchmaking
This project optionally collects additional data if you customize your profile page:
- Your display name
- Your profile picture
- Your pronoun display badges
- Your profile description text
- Your page theme

How does this project protect your password?

As previously stated, your password is encrypted before it is ever stored by this project.
This is done through hashing, or a mathematical function that is difficult to reverse.
Your hashed password would be near impossible to deduce your actual password from,
but only your actual password could produce that same hashed password.
Each time you log in, your password is compared against the stored hash to verify your identity.

How is other data stored by this project?

User data besides your password is not encrypted by this project.
This information is stored in plaintext alongside your hashed password in server storage.

Why is your password encrypted, but not other information?

Encrypting your password is also imperative for your security from credential stuffing attacks.
If user data was ever to be maliciously retrieved from server storage,
any users who reused passwords on this site would be at risk of their accounts on other sites being breached.

The primary reason that your other data is not encrypted is because I do not own the server this project is hosted on.
This project is hosted on the Electrical Engineering and Computer Science Department's student servers.
I cannot install any tool that does not currently exist on these servers,
and some tools that would provide this security require adminstrator privileges I do not have.
I was not able to find a tool suitable for this task during the development of this project.

I can encrypt your password as you should be the only user capable of accessing that information.
Encrypting other data is more complicated, as multiple users should be capable of accessing this data.

I am not going to code a custom encryption tool for this data for an additional reason:
User transparency.
I do not want to present your data as being more secure than it actually is,
and unless I was using a tool that was used by industry professionals,
I would find it dishonest to state that your data is stored encrypted.
